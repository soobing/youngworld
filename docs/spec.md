# 영월드 맵 구상

## 개요
이 강의를 진행하면서 youngworld 라는 게임맵을 만들어서, 강의자료 및 실습 결과물을 모두 해당 맵에 올리도록 진행 예정

## 컨셉
닌텐도 동물의 숲과 같은 컨셉으로, 수업에 참여하는 5명 학생과 선생님인 나까지 총 6명이 존재하는 섬.
학생들이 수업을 들으면서 이 섬을 더 발전 시키도록 개발 할 예정이고 나는 대략적인 틀만 먼저 제공

- 브라우저로 접속 가능한 웹 게임 
- 초기 섬에는 5명의 학생, 1명의 선생님 아바타, 1명의 게스트 아바타 = 총 7명의 아바타 존재
  - 아바타 꾸미기는 각자 AI를 활용하여 코딩해서 진행할 예정이므로 최대한 간단하게 제공
  - 1명의 게스트 아바타는 그냥 뷰어 역할만 함. 다른 아바타들 처럼 무언가 컨텐츠를 생성하고 대화를 나누고 할 수 없음. 자세한 기능 명세는 개발하면서 더 고도화 할 예정이나, 애초에 권한 분리가 필요함.
  - 1명의 선생님은 admin 권한을 가짐. 
    - 학생 아바타 or 게스트 아바타를 추가해야한다면 선생님만 추가할 수 있음.
  - 로그인 기능은 따로 인증을 거치지 않는 방식으로 함.
    - 각 아바타마다 닉네임과 비밀번호를 임의로 지정한 후에 처음 접속하여 비밀번호를 설정하도록 한다.
    - 초기 학생 5명 아바타의 이름은 "학생1, 학생2, 학생3, 학생4, 학생5"로 함(가명).
    - 초기 선생 1명 아바타의 이름은 "soobing"으로 함.
    - 초기 게스트 1명 아바타의 이름은 "게스트"로 함. 게스트는 비밀번호 설정하지 않아도 되게 함. 비밀번호가 없이 들어올 수 있음.
- 맵 구성
  - 강원도 영월이라는 산골마을 컨셉임. 산과 강이 흐르는 느낌으로 작은 맵 필요
  - 학교 건물 필요
    - 학교 건물에 아바타가 입장시, 교실이 보이고 교실에는 칠판과 책상의자 그리고 뒷쪽에는 아이들의 작품을 걸어놓는 형식으로 구성
    - 칠판에는 강의자료를 붙여놓을 수 있도록 하고 그 강의자료를 클릭할 경우 html ppt로 제작된 화면이 크게 보일 수 있도록 함.
## 필요한 기능
- 아바타 비밀번호 변경 기능 (초기 비밀번호는 1234로 세팅, 숫자로만 비밀번호 설정 가능하도록 함)
- admin(선생님)에서 아바타 추가 가능 기능
- 선생, 학생 아바타는 기본적으로 핸드폰 소지
  - 선생님이 학생들 전체 또는 개인에게 문자, 투표, 설문을 진행할 수 있도록 함.
  - 선생님이 예를들면 설문을 만들어 전송한다면, 학생들이 이후에 접속시 핸드폰 알람이 울리고 해당 내용을 확인하거나 답할 수 있도록 함. 계속해서 확인하지 않으면 주기적으로 알람이 뜨도록 함.

# 개발 진행 중 추가·변경된 명세 (상세)

> 위 "구상"은 초기 아이디어이고, 아래는 실제 개발을 진행하며 지시·구현한 기능들을
> **아주 상세히** 정리한 확정 명세다. (핵심 원칙: 모든 권한·집계는 **서버에서 강제**하고,
> 클라이언트의 버튼 숨김은 장식일 뿐이다.)

## 0. 용어·역할
- **admin(선생님, `soobing`)**: 모든 권한(`*`). 아바타/자료/미션 관리, 설문·미션 현황 열람.
- **student(학생)**: 이동·설정·쪽지·설문 응답·미션 완료. 자기 것만 조회.
- **guest(게스트)**: 관람만. 설정·쪽지·설문·미션·우편함 전부 없음(비번 없이 입장).

## 1. 내 캐릭터 설정 팝업 (⚙️ / 맵에서 내 캐릭터 클릭)
맵에서 **자기 자신 아바타를 클릭**(또는 우하단 ⚙️ 버튼)하면 픽셀 게임풍 설정 창이 열린다.
게스트는 이 창이 없다. 창은 로그인 화면과 같은 픽셀 디자인 언어(검은 외곽선·하드 베벨·나무판·픽셀 폰트).

### 1.1 탭 구조 (2차 개편 결과)
상위 탭 **4개**로 정리하고, 설문·미션은 **내부 서브탭**으로 만들기↔결과를 오간다.
- **내 정보** — 닉네임 변경 + 비밀번호 변경(통합)
- **쪽지** — 친구/선생님에게 쪽지 보내기
- **설문** — 서브탭 [만들기] / [결과]
- **미션** — 서브탭 [만들기(선생님만)] / [결과]
  - 미션 탭은 **모두에게 보이되**, "만들기"는 선생님 전용. 학생은 "결과"만 보고(서브탭 바 숨김) **자기 미션·완료여부만** 본다.

> 서브탭 라벨은 "현황"이 아니라 **"결과"** 로 확정.

### 1.2 내 정보 탭
- **닉네임 변경(1~12자, 중복 불가)**: 저장 시 서버가 `player:renamed`를 전원에게 방송 →
  맵의 이름표·전시 이름이 실시간 갱신. 중복이면 `DUP_NICK` 오류.
- **비밀번호 변경(내 정보 탭에 통합)**: 기존 맵의 🔒 버튼·별도 모달을 제거하고 닉네임 아래에
  기존/새 비밀번호 입력 + "🔒 비밀번호 변경"을 배치. 숫자 4자리 이상만 허용.

## 2. 우편함 (구 "받은 쪽지함")
"받은 쪽지함"에서 **쪽지·설문·미션을 모두 보므로 이름을 "우편함"으로 변경**(📮).
쪽지·설문·미션 아이템이 한 목록에 함께 뜬다. 아이템별 동작:
- **쪽지(sms)**: 본문 표시 + "확인"(읽음) + "↩ 답장"(보낸 사람에게 sms로 회신, 제목 `RE:`).
- **설문(survey)**: 질문마다(객관식=라디오 / 주관식=텍스트) 응답 후 제출.
  - **받은 설문의 질문은 굵게(strong) 표시.**
  - 이미 응답한 설문은 `질문` 다음 줄에 **"ㄴ 내 답"** 형태로 개행 표시.
- **미션(mission)**: 할 일 + 기한 + 임박 배지 + "✔ 완료 체크". 완료하면 "✅ 완료했어요".

---

## 3. 쪽지 (peer message)
- 학생·선생님 서로 보낼 수 있음(게스트 제외, 자기 자신 제외).
- 받는 사람: 개별 지정 또는 **"📢 모두에게(게스트 제외)"** 전체 발송.
- 다른 사람 아바타를 클릭하면 그 사람에게 쪽지 쓰기 창이 바로 열린다.
- 서버 이벤트 `peer:send { recipientNickname | target:'all', body }`.

---

## 4. 설문 조사 (survey) — "투표"에서 확장
초기엔 1문항 "투표"였으나, **다중 질문 설문**으로 확장하고 poll 타입 코드는 제거했다.

### 4.1 만들기 (설문 탭 > 만들기)
- **질문 여러 개**: 기본 1개 + "＋ 질문 추가" / 각 질문 삭제(×).
- 질문마다 **유형 선택 셀렉트**: **객관식** / **주관식**(구글 설문지처럼 자유 서술).
  - 유형 셀렉트는 질문 헤더의 **× 버튼 왼쪽**에 붙이고, 크기를 작게.
  - 객관식: 보기 2개 이상(쉼표 구분). 주관식: 보기 없음.
- **대상**: 전체(게스트 제외) / 개인.
- **공개 여부**: 공개(설문 결과 목록에 노출) / 비공개(나만 결과 열람).
- 제목(선택).

### 4.2 결과 (설문 탭 > 결과) — 목록 → 상세(depth 1)
- **목록 카드**: 제목 + `설문 생성자: 이름 · 응답 N/M명 · 질문 K개`(+ 비공개면 🔒).
  - (초기의 "내가 만듦" 라벨 UI는 제거하고 **"설문 생성자: 이름" 텍스트**로 바꿈.)
- **상세**: 카드 클릭 시 진입. 상단 "← 목록".
  - **응답 보기 셀렉트**로 시점 전환:
    - **📊 통계**: 질문별 집계(객관식=막대·%·1위 강조, 주관식=응답 텍스트 목록). 내 답은 ✔.
    - **👤 응답자별**(생성자만) / **🙋 내 응답**(응답자): 그 사람의 답을 통계와 **같은 질문-박스 레이아웃**으로.
  - **생성자 전용 설정**(상세에만, `s.mine`):
    - **공개 상태** 셀렉트 + **저장** 버튼 → `survey:setPublic`. 저장 시 **토스트** 안내(재렌더에도 안 지워지게).
    - **결과 공유** 셀렉트 + 저장 → `survey:setShareResults`.

### 4.3 노출·권한 규칙 (서버 강제)
- **목록 노출**: `공개 설문` + `내가 만든 설문`(비공개 포함)만. 남의 비공개는 안 보임.
  (기능 추가 전 만든 설문은 공개로 간주 = `payload.public !== false`.)
- **개별 응답(누가 무엇을 답했나)**: **항상 생성자만**. 서버가 비생성자에겐 `responses`를 아예 안 내려줌.
- **통계(집계)**: 생성자, 또는 생성자가 **"결과 공유"를 켠 경우**의 비생성자. 권한 없으면 `counts=null`.
- **응답자**: 자기 답(myAnswers)만.
- **실시간**: 역할별 데이터가 달라 결과를 브로드캐스트하지 않고 `survey:changed`(변경 신호)만 보내
  각자 `survey:mine`으로 자기 관점 재요청.

## 5. 미션 (mission) — 선생님이 내는 TODO
쪽지와 유사하나 "해야 할 일 + 기한 + 완료 체크"가 있는 과제. `phone_messages`(type='mission') 재사용.

### 5.1 만들기 (선생님만)
- **해야 할 일**(textarea), **기한**(date), **대상**(전체/개인), 제목(선택).
- 서버 `mission:create`. 대상에게 즉시 알림(`phone:new`).

### 5.2 도착·알림
- 로그인 시 **미완료 미션이 알람으로** 뜨고, 우편함에서 쪽지·설문과 함께 조회.
- **맵 알림 아이콘은 쪽지(💌)와 구분되는 별도 아이콘 💥**(마감 임박 시 ⏰). 둘 다면 나란히.
- **핵심 설계**: 미션의 `is_read`(=완료 여부)로 두어, **완료 전까지 계속 알람**이 뜬다.

### 5.3 완료 (학생)
- 우편함에서 "✔ 완료 체크" → `mission:complete` → 응답 저장(`answered_at` 세팅). 배지 감소.

### 5.4 결과 (미션 탭 > 결과)
- **선생님(생성자)**: 내가 낸 미션 목록 → 상세. 진행률 막대·**%**·**완료자/미완료자 이름 칩**.
- **학생**: **자기가 받은 미션 + 자기 완료여부만**(남의 완료율·명단은 서버가 안 줌). `mission:received`.
- 권한: 완료자 명단·%는 **생성자만**. 학생의 전체 현황(`mission:mine`) 접근은 거부.

### 5.5 마감 임박 리마인더 (아이디어 구현)
- 서버가 마감일로 **임박도**를 실시간 계산: `later(여유) → soon(3일 이내) → today(하루 이내) → overdue(지남)`.
- 미완료 미션은 로그인 시 + 주기적으로 `mission:remind`를 보내고, **오늘/지남**이면 **토스트로 계속 상기**시키며 맵 아이콘을 ⏰로 강조. 완료하면 멈춤.

## 6. 알림 UI
- **맵 머리 위 아이콘**(통통 튐, 클릭 → 우편함): 💌(안읽은 쪽지/설문) + 💥/⏰(미완료 미션).

  (설문 공개상태 저장, 관리 완료, 미션 마감 리마인더 등에 사용.)

## 7. 맵 · 씬 변경
### 7.1 섬(IslandScene)
- 라벨: `흙밭(빈 땅)` → **밭**, `마을 광장` → **동강 둔치**(라벨 위치 한 줄 아래로).
- **학교 건물 축소**(3×3 → 3×2), 지붕 처마를 넉넉히 빼 **벽 크림색 삐져나옴(살색) 보수**.
- **"← 학교" 푯말**: 동강 다리를 건넌 직후(강 북쪽) 학교로 꺾이는 길목으로 이동, 방향 `학교 →` → `← 학교`.

### 7.2 교실(ClassroomScene)
- **맵을 화면(800×600)보다 크게(25×19=800×608)** 잡아 **하늘색 여백 제거**(꽉 채움).
- **책상**: 회색 15개 → **나무 책상**. 배치는 앞줄 2·뒷줄 3이되 **일자(정면)**.
  **학생 수에 맞춰 자동 증감**(한 줄 최대 3, 가운데 정렬). 의자 포함.
- **캐릭터 z-index를 책상 위**로(책상에 앉은 느낌).
- **바닥**: 아이보리 → 나무마루. 작품 구역 러그: 핑크 → **세이지(민트)**.
- **문**: 보라색 타일 → **나무 문짝**(문틀·2짝·손잡이) + "🚪 나가기".
- **작품 구역**: 축소 + 큰 액자 제거 → 헤더 "작품" + **학생 이름만**(색 점 + 닉네임) 나열. 이름 클릭 → 갤러리.

## 8. 작품 갤러리 (교실)
- 학생별로 **4개 작품**을 전시: `👋 자기소개 · ⭐ 나의 꿈은? · 🎮 5년 뒤 나의 미래(게임) · 📚 10년 뒤 나의 미래(웹툰)`.
- 교실의 학생 이름을 클릭하면 **픽셀 팝업**이 열리고, ◀ ▶로 학생을 넘기며 **2×2 카드**로 4작품을 본다.
- 작품이 있으면 카드 클릭 → 전체화면(iframe), 없으면 "전시 준비 중".
- 데이터: `gallery_works`(author_id=학생, slot=카테고리 0~3) 재사용. 스키마 변경 없음.

## 12. 보안 하드닝 (공개 서버 호스팅 대비)
- **비밀번호**: 평문 저장 → **`crypto.scrypt` 해싱**(salt+hash, 형식 `scrypt$salt$hash`). 구 평문은 로그인 시 자동 해시 승격.
- **세션**: 토큰 **14일 만료** + 서버 시작 시 만료 세션 정리.
- **로그인 무차별 대입 방어**: IP+닉네임별 **8회 실패 → 10분 잠금**(429).
- **XSS 차단**: 강의자료·책장 문서 URL을 **`/lectures/`·`/guides/` 아래 `.html` 만 허용**(외부·경로탈출·`javascript:` 차단). iframe 임의 HTML 로드 불가.
- 사용자 입력(닉네임·쪽지·설문답·미션내용)은 `textContent`/`escapeHtml`로 이스케이프.
- 프록시 뒤 배포 시 `trust proxy`로 실제 IP 인식, Node는 `HOST=127.0.0.1` 바인딩(외부는 Caddy 443만).

## 14. 데이터 모델 요약
- **avatars**: id, nickname(UNIQUE), role, password(해시), must_change, color, last_x/y/scene.
- **sessions**: token(PK), avatar_id, created_at(14일 TTL).
- **phone_messages**: id, sender_id, **type('sms'|'survey'|'mission')**, title, payload(JSON), target('all'|'individual'), created_at.
  - survey payload: `{ questions:[{q, options[]}], public, shareResults }` (주관식은 options 빈 배열).
  - mission payload: `{ task, due }`.
- **phone_deliveries**: message_id, recipient_id, is_read(미션은 =완료), answered_at, UNIQUE(message,recipient).
- **phone_responses**: delivery_id, answer(JSON), created_at.
- **lecture_materials / gallery_works / guide_docs**: 칠판 자료 / 작품(작가·slot) / 책장 문서.

## 15. 주요 소켓 이벤트 요약
- 이동/씬: `player:move`·`player:moved`, `scene:enter`·`scene:ready`, `player:joined`·`left`·`renamed`.
- 설정: `me:setNickname`·`me:updated`.
- 쪽지: `peer:send`·`peer:sent`, `phone:markRead`, `phone:reply`.
- 우편함 공통: `phone:new`, `phone:alarm`, `phone:answered`.
- 설문: `peer:survey`, `survey:mine`·`survey:changed`, `survey:setPublic`, `survey:setShareResults`, `phone:respond`.
- 미션: `mission:create`, `mission:complete`, `mission:mine`(생성자)·`mission:received`(학생), `mission:changed`, `mission:remind`.
- 관리(admin): `admin:addAvatar`·`admin:deleteAvatar`, `admin:postMaterial`, `admin:addGuide`·`admin:deleteGuide`, `admin:addWork`.
- 갱신 방송: `blackboard:update`, `gallery:update`, `guides:update`.
