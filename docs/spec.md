# 영월드 맵 구상

## 개요
이 강의를 진행하면서 youngworld 라는 게임맵을 만들어서, 강의자료 및 실습 결과물을 모두 해당 맵에 올리도록 진행 예정

## 컨셉
닌텐도 동물의 숲과 같은 컨셉으로, 수업에 참여하는 5명 학생과 선생님인 나까지 총 6명이 존재하는 섬.
학생들이 수업을 들으면서 이 섬을 더 발전 시키도록 개발 할 예정이고 나는 대략적인 틀만 먼저 제공

- 브라우저로 접속 가능한 웹 게임 
- 초기 섬에는 5명의 학생, 1명의 선생님 아바타, 1명의 게스트 아바타 = 총 7명의 아바타 존재
  - 아바타 꾸미기는 각자 AI를 활용하여 코딩해서 진행할 예정이므로 최대한 간단하게 제공
  - 1명의 게스트 아바타는 그냥 뷰어 역할만 함. 다른 아바타들 처럼 무언가 컨텐츠를 생성하고 대화를 나누고 할 수 없음. 자세한 기능 명세는 개발하면서 더 고도화 할 예정이나, 애초에 권한 분리가 필요함.
  - 1명의 선생님은 admin 권한을 가짐. 
    - 학생 아바타 or 게스트 아바타를 추가해야한다면 선생님만 추가할 수 있음.
  - 로그인 기능은 따로 인증을 거치지 않는 방식으로 함.
    - 각 아바타마다 닉네임과 비밀번호를 임의로 지정한 후에 처음 접속하여 비밀번호를 설정하도록 한다.
    - 초기 학생 5명 아바타의 이름은 "학생1, 학생2, 학생3, 학생4, 학생5"로 함(가명).
    - 초기 선생 1명 아바타의 이름은 "soobing"으로 함.
    - 초기 게스트 1명 아바타의 이름은 "게스트"로 함. 게스트는 비밀번호 설정하지 않아도 되게 함. 비밀번호가 없이 들어올 수 있음.
- 맵 구성
  - 강원도 영월이라는 산골마을 컨셉임. 산과 강이 흐르는 느낌으로 작은 맵 필요
  - 학교 건물 필요
    - 학교 건물에 아바타가 입장시, 교실이 보이고 교실에는 칠판과 책상의자 그리고 뒷쪽에는 아이들의 작품을 걸어놓는 형식으로 구성
    - 칠판에는 강의자료를 붙여놓을 수 있도록 하고 그 강의자료를 클릭할 경우 html ppt로 제작된 화면이 크게 보일 수 있도록 함.
## 필요한 기능
- 아바타 비밀번호 변경 기능 (초기 비밀번호는 1234로 세팅, 숫자로만 비밀번호 설정 가능하도록 함)
- admin(선생님)에서 아바타 추가 가능 기능
- 선생, 학생 아바타는 기본적으로 핸드폰 소지
  - 선생님이 학생들 전체 또는 개인에게 문자, 투표, 설문을 진행할 수 있도록 함.
  - 선생님이 예를들면 설문을 만들어 전송한다면, 학생들이 이후에 접속시 핸드폰 알람이 울리고 해당 내용을 확인하거나 답할 수 있도록 함. 계속해서 확인하지 않으면 주기적으로 알람이 뜨도록 함.

# 추가 구현 명세 (초기 구상 이후 지시·확정 사항)

> 아래는 초기 구상(위)에서 출발해 개발 과정에서 **추가로 지시하고 확정한 기능**을 상세히 정리한 것이다.
> 원래 "핸드폰"으로 부르던 개념은 개발 중 **"우편함"** 으로 이름이 바뀌었고(쪽지·설문·미션을 한곳에서 봄),
> "투표"는 다중 질문이 가능한 **"설문"** 으로 통합·확장되었으며(별도 `poll` 타입은 코드/데이터에서 완전히 제거),
> 자기 캐릭터 클릭으로 여는 **"설정"** 창이 사용자 상호작용의 중심이 되었다.

## 1. 내 캐릭터 설정 (설정 창)
- 맵에서 **자기 자신 캐릭터를 클릭**(또는 우하단 ⚙️ 버튼)하면 픽셀 게임풍 설정 창이 팝업으로 뜬다. 게스트는 정체성이 없어 사용 불가.
- 다른 사람 캐릭터를 클릭하면 그 사람에게 **쪽지 쓰기** 창이 바로 열린다.
- 창 제목은 "⚙ 설정". **상위 탭 4개**로 구성: `내 정보` · `쪽지` · `설문` · `미션`.
  - `설문`·`미션` 탭은 내부에서 **서브탭(만들기 / 결과)** 으로 전환한다(탭이 너무 많아지지 않게).
  - `미션` 탭은 모두에게 보이되, **"만들기" 서브탭은 선생님만**(학생은 "결과"만).
- **내 정보 탭**
  - 닉네임 변경(1~12자, 중복 불가). 저장 즉시 맵의 이름표가 실시간으로 바뀜(모든 접속자에게 방송).
  - **비밀번호 변경**을 이 탭 아래에 통합(기존 비번 + 새 비번). 맵의 별도 🔒 버튼·모달은 제거.

## 2. 우편함 (구 "받은 쪽지함")
- 우하단 📮 버튼으로 열며, **쪽지·설문·미션을 한 곳에서** 조회. 게스트 제외 모두 사용.
- **맵 알림(머리 위 아이콘)**: 안읽은 쪽지/설문이 있으면 💌, 미완료 미션이 있으면 💥(마감 임박 시 ⏰). 둘 다면 나란히 표시되고, 클릭하면 우편함이 열린다.
- 새 항목 도착 시 소리·흔들림·배지 숫자. 로그인 시 안읽은 항목을 알람으로 받고, 미확인이 있으면 주기적으로 재알람.

## 3. 쪽지
- 학생·선생님 누구나 서로에게 쪽지를 보낼 수 있음(게스트 제외). 받는 사람: **개인 또는 "📢 모두에게"**.
- 받은 쪽지는 우편함에서 **확인(읽음)** 하거나 보낸 사람에게 **답장**(RE:) 가능.

## 4. 설문 (구 "투표"를 통합·확장)
- **설문 만들기**(설문 탭 > 만들기 서브탭)
  - 대상(전체/개인), 공개 여부(공개/비공개), 제목(선택).
  - **질문 여러 개**: 기본 질문 1개 + `＋ 질문 추가`(삭제 ×). 질문마다 **객관식/주관식** 선택.
    - 객관식: 보기 2개 이상(쉼표 구분). 주관식: 자유 서술.
- **응답(우편함)**: 질문마다 객관식은 라디오, 주관식은 텍스트로 답하고 제출. 응답 후엔 질문(굵게) + 개행 `ㄴ (내 답)` 형태로 표시.
- **설문 결과**(설문 탭 > 결과 서브탭): 목록 → **상세(depth 한 단계)**.
  - 상세 상단 **"응답 보기" select**: `통계`(전체 집계) 또는 **응답자별**(그 사람 답을 같은 레이아웃으로).
  - 객관식은 득표 막대(+%, 1위 강조, 내 답 ✔), 주관식은 응답 텍스트 목록.
- **공개/비공개 및 결과 공유 (권한 분리, 서버에서 강제)**
  - **공개 여부**: 공개 설문만 남의 "결과" 목록에 노출(내가 만든 건 항상). 생성 시 지정하고, 이미 만든 설문도 **생성자가 상세에서 select+저장으로 변경** 가능.
  - **개별 응답(누가 뭘 답했나)은 항상 생성자만** 볼 수 있음.
  - **통계(집계)**는 생성자, 또는 생성자가 **"응답 결과 공유하기"를 켠 경우**에만 다른 사람이 볼 수 있음(공유해도 개별 응답은 여전히 생성자 전용).
  - **응답자는 자기 자신의 답만** 확인.
  - 공개/공유 상태 편집 UI는 **생성자에게만** 보이고 저장 시 토스트로 안내.

## 5. 미션 (TODO 성격)
- **미션 만들기(선생님만)**: 대상(전체/개인), 제목(선택), **해야 할 일(textarea)**, **기한(날짜)**.
- 도착하면 우편함에서 쪽지·설문과 함께 조회하되, **맵 알림 이모티콘은 쪽지(💌)와 다른 💥**(마감 임박 ⏰).
- **완료 처리**: 학생이 우편함에서 **"✔ 완료 체크"** 하면 완료로 기록(완료 전까지는 계속 알림 대상).
- **미션 현황(선생님)**: 목록(진행률 막대·%) → 상세(✅ 완료자 이름 / ⏳ 미완료자 이름 칩). **전체 완료율·완료자 명단은 생성자만**.
- **학생의 미션 결과**: 미션 탭 "결과"에서 **자기가 받은 미션과 자기 완료 여부만**(남의 완료율·명단은 서버가 아예 내려주지 않음).
- **마감 임박 리마인더**: 서버가 마감까지 남은 시간으로 임박도(여유→3일내→오늘→지남)를 실시간 계산. 미완료 미션은 로그인 시·주기적으로 리마인더를 보내고, 오늘/지남이면 토스트로 계속 상기시키며 맵 아이콘을 ⏰로 강조.

## 6. 교실 (내부 리모델링)
- **책상**: 학생 수에 맞춰 **자동으로 개수 증가**(한 줄 최대 3, 가운데 정렬). 회색 → **나무 책상 + 의자**, 정면(일자)으로 칠판을 바라봄. 캐릭터가 책상 **위(z-index)** 로 그려져 "앉은" 느낌.
- **바닥**: 아이보리 → 따뜻한 나무마루 텍스처.
- **나가는 문**: 보라색 타일 → 나무 문짝 + "🚪 나가기" 안내.
- **작품 갤러리**: 교실 오른쪽 세이지색 러그 구역. 큰 액자 없이 **학생 이름만**(색 점 + 닉네임) 세로 나열, 헤더는 "작품". 이름을 클릭하면 그 학생의 작품 팝업이 열림.
  - 작품은 학생별 **4가지 카테고리**: `👋 자기소개` · `⭐ 나의 꿈은?` · `🎮 5년 뒤 나의 미래(게임)` · `📚 10년 뒤 나의 미래(웹툰)`. 팝업에서 ◀▶로 학생을 넘기며 2×2 카드로 보고, 작품이 있으면 클릭해 전체화면 열람(없으면 "전시 준비 중").
- **책장(How-to 안내 문서)**: 교실 왼쪽 아래 책장을 클릭하면 문서 목록 팝업 → 항목 클릭 시 전체화면 열람. 칠판(강의자료)과 별개의 참고 자료 보관함. 기본 문서: **GitHub 가입하는 법 · Claude 가입하는 법**(추가 가능).
- **하늘색 여백 제거**: 교실 맵을 캔버스보다 크게(25×19) 잡아 레터박스 없이 꽉 채움.

## 7. 섬(야외) 맵 다듬기
- 학교 건물 축소 및 지붕 밖으로 삐져나오던 벽 모서리(“살색”) 보수. 문·진입 위치는 유지.
- 지명 라벨: `흙밭(빈 땅)` → **밭**, `마을 광장` → **동강 둔치**(라벨 위치 한 줄 아래로).
- 학교 이정표: 동강 다리를 건넌 북쪽 길목으로 옮기고 방향을 `← 학교`로.

## 8. 아바타 색상 · 성별
- 아바타 색을 **DB(`avatars.color`) 단일 출처**로 통일 → 로그인 카드 점 · 인게임 미니미 · 갤러리 이름 점이 모두 동일.
- (성별 참고 정보에 따라 색을 배치했으나) 공개 저장소에는 **개인정보(실명·성별)를 남기지 않도록** 학생 이름을 가명(`학생1~5`)으로, 성별 주석은 제거.

## 9. 로그인 · 인트로
- 타이틀 2줄 "Youngworld / 입장하기", 서브 "AI와 함께하는 진로 멘토링".
- 캐릭터 카드 순서: `선생님 / 게스트` → `학생1 / 학생2` → `학생3 / 학생4` → `학생5`.
- 카드 영역 스크롤 제거(마지막 카드 테두리 잘림 방지).

## 10. 영월드 관리센터 (구 "선생님 관리")
- 선생님(admin)만 여는 관리 패널. 기존 "핸드폰 보내기" 기능은 삭제(설문/미션은 설정 창에서 냄).
- 기능: **아바타 추가 / 아바타 삭제**(선생님은 삭제 불가, 접속 중이면 강제 종료) / **강의자료 게시(칠판)** / **책장 문서 추가·삭제**.

## 11. 알림(토스트) UI
- 화면 **왼쪽 위**에 뜨고, 여러 개가 나오면 아래로 쌓이며 위로 올라오는 스택 방식.

## 12. 데이터/타입 정리
- 별도 `poll`(투표) 메시지 타입은 코드·DB에서 완전히 제거하고 `survey`(설문)로 일원화. 기존 투표는 1문항 설문으로 정규화되어 그대로 열람/응답 가능.
- 우편함·설문·미션은 기존 메시지 인프라(`phone_messages`/`phone_deliveries`/`phone_responses`)를 재사용(스키마 변경 최소화). 미션은 `type='mission'`, 완료 여부를 응답으로 기록.

## 13. 공개 배포용 보안 (아무나 접속 가능한 공개 서버 전제)
- **비밀번호 해싱**: Node `crypto.scrypt`(salt+hash) 저장. 기존 평문은 로그인 시 자동으로 해시 승격. (초기 비번 1234, 숫자 4자리 이상 규칙은 유지)
- **세션 만료**: 로그인 토큰 14일 TTL, 서버 시작 시 만료 세션 정리.
- **로그인 무차별 대입 방어**: IP+닉네임별 8회 실패 시 10분 잠금.
- **XSS 차단**: 강의자료·책장 문서 URL을 같은 서버의 `/lectures/`·`/guides/` 아래 `.html`만 허용(외부·경로탈출·`javascript:` 차단). 사용자 입력 텍스트는 이스케이프.
- **개인정보 보호**: DB 파일은 git 미포함(.gitignore), 소스에 실명 없음. 공개 리포는 실명 히스토리 없이 새로 생성.

## 14. 배포 구성
