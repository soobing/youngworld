1협업하며 코드 관리하기

프로그램은 글자로 된 코드(code) 수천 줄로 이뤄져요. 여럿이 고치다 보면 금세 뒤죽박죽이 되죠. 그래서 개발자들은 버전 관리(형상 관리) 도구를 씁니다.

Git — 코드의 타임머신

Git 은 코드가 언제·어떻게 바뀌었는지 기록(버전)을 남겨줘요. 실수해도 예전 버전으로 되돌릴 수 있죠.

📄 비유: 발표 자료를 고칠 때 발표 → 발표_최종 → 발표_진짜최종 → 발표_찐찐최종처럼 파일을 자꾸 새로 만든 적 있죠? Git은 이런 이름 놀이 없이, 하나의 파일에 바뀐 기록을 순서대로 저장해 줘요. 예전 버전으로 되돌리기도 한 번에!

GitHub — 코드를 함께 두는 인터넷 공간

GitHub 는 내 컴퓨터의 코드를 인터넷에 올려 여럿이 함께 보고 나누는 곳이에요. (Git = 도구, GitHub = 코드를 모아두는 웹사이트!)

📁 비유: 발표 자료를 구글 드라이브에 올려 친구와 함께 보고 고치는 것과 비슷해요. (GitHub는 '코드 전용' 드라이브인 셈!)
🔗 여러분이 GitHub에 가입하고 Username을 쪽지로 공유한 것도, 바로 이 개발자들의 세계로 들어온 첫걸음이에요!

브랜치(branch) — 원본을 지키는 '가지' 🌿

원본을 바로 고치면 위험해요. 그래서 branch(가지)를 쳐서 복사본에서 마음껏 작업하고, 잘 되면 원본에 합칩니다.

🌿 비유: 숙제 파일을 복사해서 '연습용 사본'을 만들고, 거기서 이것저것 고쳐보는 것과 같아요. 원본은 안전하니 마음 놓고 실험하고, 마음에 들 때만 원본에 반영해요.

PR(Pull Request) — "합쳐도 될까요?" 검토 요청

가지에서 바꾼 걸 원본에 합치기 전에, Pull Request(PR)로 "이렇게 바꿨는데 합쳐도 될까요?" 하고 동료에게 검토를 요청해요. 함께 확인하니 실수가 줄어요.

📝 비유: 숙제를 바로 붙이는 게 아니라, 선생님께 먼저 검사받고(리뷰) 통과하면 합치는 것과 같아요.

2오픈소스 문화

오픈소스(Open Source)는 프로그램의 설계도(소스 코드)를 공개해서, 누구나 보고·쓰고·고치고·나눌 수 있게 하는 문화예요.

🔍 열려 있어요

어떻게 만들었는지 보고 배우고 따라 할 수 있어요.

🤝 함께 만들어요

여럿이 기여(contribution)해서 점점 좋아져요.

🎁 나눌수록 커져요

내가 나누면 다른 사람이 이어서 발전시켜요.

🙏 존중과 감사

남의 코드를 쓸 땐 감사하고 규칙(라이선스)을 지켜요.

🌏 우리가 만드는 영월드 게임도 전 세계 개발자들이 무료로 공개한 오픈소스 도구(예: 게임 라이브러리 Phaser) 위에서 만들어졌어요!

3AI로 코딩하는 두 가지 방법 — 채팅 vs 에이전트

요즘 AI는 코딩을 도와줘요. 그런데 쓰는 방법이 두 가지예요. 둘 다 속은 같은 AI(LLM)인데, 차이는 '어떻게 쓰느냐'죠.

🍳 비유: 채팅은 옆에서 레시피를 알려주는 친구, 에이전트는 주방에 들어와 같이 요리해 주는 친구예요.
🧠 알아두기: 코딩 에이전트도 속은 LLM이 움직여요. '채팅'은 그 AI가 말로 답하는 것, '에이전트'는 그 AI에게 손(도구)을 쥐여줘 직접 파일을 고치게 한 거예요.
구분LLM 채팅 서비스코딩 에이전트
예시ChatGPT, Claude 앱Claude Code, Codex
일하는 곳웹/앱 대화창내 실제 프로젝트 폴더 안
코드 반영내가 복사해서 붙여넣기AI가 파일을 직접 수정·실행
잘하는 일설명, 아이디어, 짧은 코드여러 파일을 아우르는 실제 작업

4이번 실습에서 Claude를 고른 이유

이번 수업에서는 코딩 에이전트 Claude(Claude Code)로 실제 게임을 함께 만들어요. 이유는 이래요.

  1. 프로젝트 전체를 이해해요. 파일 하나가 아니라 게임 전체를 읽고, 맥락에 맞게 고쳐 줘요.
  2. 읽기 → 고치기 → 실행까지 한 번에 해요. 결과를 바로 확인하며 배울 수 있어요.
  3. 안전장치가 있어요. 위험하거나 되돌리기 어려운 일은 먼저 물어보고 진행해서, 처음 배우는 사람도 안심돼요.
  4. 한국어 설명이 자연스러워요. 왜 그렇게 고쳤는지 친절히 알려줘요.
  5. 이 게임(영월드)이 실제 증거예요. 여러분이 지금 하고 있는 이 게임도 Claude와 함께 만들었어요!
💡 도구는 도구일 뿐이에요. 가장 중요한 건 무엇을 왜 만들지 생각하는 사람, 바로 여러분이에요. AI는 그 생각을 빠르게 도와주는 친구랍니다.

5우리가 만드는 게임은 어떻게 움직일까?

영월드는 브라우저 게임이에요. 설치 없이 인터넷 주소만 열면 바로 실행되죠. 그 안을 들여다볼게요.

① 브라우저 게임 & 라이브러리(Phaser)

웹페이지를 만드는 HTML·CSS·JavaScript로 만들어요. 캐릭터를 그리고 움직이게 하는 게임 부분은 Phaser ↗라는 게임 라이브러리가 도와줘요.

라이브러리(library)가 뭐냐면 — 남이 미리 잘 만들어 둔 편리한 도구 모음이에요. 캐릭터 움직임·충돌·애니메이션 같은 걸 처음부터 다 만들지 않고, Phaser에서 가져다 쓰는 거죠.

🧱 비유: 라이브러리는 레고 블록이나 요리 밀키트 같아요. 바닥부터 다 만들지 않고, 잘 만들어진 재료를 조립해서 훨씬 빠르게 완성해요.

② 앞쪽과 뒤쪽 — 프론트엔드 & 백엔드

게임은 크게 두 부분으로 나뉘어요. 눈에 보이는 앞쪽과 뒤에서 조용히 일하는 뒤쪽이죠.

🎨 프론트엔드 (앞쪽)

화면·버튼·캐릭터처럼 눈에 보이고 만지는 부분. 우리 게임에선 클라이언트 쪽이에요.

⚙️ 백엔드 (뒤쪽)

저장·전달·규칙처럼 뒤에서 일하는 부분. 우리 게임에선 서버·DB 쪽이에요.

🔎 그래서 앞쪽 작업을 프론트엔드, 뒤쪽 작업을 백엔드라고 불러요. 둘 다 잘하면 풀스택!

③ 세 친구의 팀워크 — 클라이언트 · 서버 · DB

🖥️ 클라이언트

내 화면에 게임을 그리고 내 조작을 받아요. 친구마다 각자 있어요.

🗄️ 서버

모두를 연결하는 중심. 소식을 정리해 모두에게 전달해요.

💾 데이터베이스(DB)

아바타·쪽지·작품을 저장. 껐다 켜도 정보가 남아요.

⚡ 실시간(Socket.io)

클라이언트와 서버가 계속 연결돼 소식을 바로 주고받아요.

④ 함께 노는 비결 — 실시간 멀티플레이

📞 비유: 서버는 전화 교환실이에요. 내가 움직이면 서버에 알리고, 서버가 그 소식을 친구들 화면에 전해 줘서 실시간으로 같이 보이게 돼요.
🏝️ 여러분이 섬을 걷고, 쪽지를 보내고, 작품을 거는 모든 순간이 바로 이 구조로 돌아가고 있어요!

6코드가 인터넷에 올라가기까지 (배포)

완성한 코드는 어떻게 진짜 인터넷 주소에서 친구들이 접속하게 될까요? 이 과정을 배포(deploy)라고 해요.

🎬 비유: 유튜브에 영상을 업로드하면 전 세계 누구나 볼 수 있죠? 배포도 똑같아요. 내 코드를 인터넷에 올리면(배포), 누구나 주소로 접속해 즐길 수 있게 돼요.
⚡ 1번에서 배운 GitHub가 여기서도 쓰여요. 코드를 push 하면 → 자동 배포 → youngworld-ai.com 에 반영. 이렇게 배운 게 다 연결돼요!
